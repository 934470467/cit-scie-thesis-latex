# LaTeX User Manual PRD

## 业务背景
现有的 3 个 LaTeX skill 文件信息冗余度高，不够精简，导致新用户使用时消耗过多的 token。
为解决此问题，需要对 skill 进行极致压缩，并输出一本基于金字塔原理编排的《LaTeX 使用手册》供最终查阅。

## User Stories
1. **作为 AI Agent（或新用户）**，我希望技能文件尽量短小精悍，只有“如果是 A，执行 B”，这样可以省 Token 且减少幻觉。
2. **作为开发者**，我希望整个构建过程一键自动化，有一个 `build_manual.sh` 帮我自动跑四次 LaTeX 编译过程。
3. **作为开源贡献者**，我希望所有的修改与新增手册都能被干净地 Commit 到本地仓库中，随时可以提 PR 给模板原作者。

## 原子化任务 (见 prd.json)
1. 压缩定制修改技能
2. 压缩入口决策技能
3. 压缩工作流核心技能
4. 建立手册 LaTeX 骨架
5. 填充章节内容
6. 提供自动化 Shell 脚本
7. 跑通编译流程并 Commit
